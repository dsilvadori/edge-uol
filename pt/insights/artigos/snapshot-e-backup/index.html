Snapshot e backup

Tecnologia

Virtualization Technologies

Postado em: 16/12/2022
Tempo de leitura: 4 minutos
Por: Welington Camara

O que é um snapshot?

Um snapshot como é remetido, trata-se de uma fotografia do momento atual de uma máquina virtual. Esse snapshot é salvo no próprio espaço de armazenamento da caixa física onde se contém as máquinas virtuais. O snapshot trata-se de cópia de metadados e contém apontamentos que são utilizados para uma possível recuperação no caso de uma perda parcial ou total da máquina virtual.

O que é o backup?

A palavra backup no mundo de tecnologia significa uma cópia de segurança de dados de um dispositivo de armazenamento (tablet, celular, computadores e outros meios de armazenamento). Hoje em dia existem diversas ferramentas com soluções para trabalhar com backup de grandes fabricantes, alguns deles abaixo:

  2. Unitrends
  3. Veeam Backup & Replication
  4. Bacula Enterprise
  5. Acronis Cyber Backup
  6. Cohesity
  7. MSP360
  8. IBM Spectrum Protect
  9. Dell EMC Data Protection Suite
  10. Backblaze

Para solução de backup devemos sempre pensar no CDP (Continuous Data Protection) que englobam dois indicadores importantes conhecidos como RPO (Recovery Point Objective) e RTO Recovery Time Objective).

O RPO (Recovery Point Objective) é o período entre um backup e outro, por exemplo, uma semana, um dia ou horas. O RTO (Recovery Time Objective) é o tempo de recuperação de um backup, por exemplo, quando se pergunta: “qual é o tempo de restore?” - esse é o RTO.

Tratando-se do CDP (Continuous Data Protection), podemos explanar que significa um planejamento contínuo de backup para garantir a integridade dos dados de forma estruturada e eficaz.

Diferença entre backup e snapshot

Apesar de muitas pessoas entenderem que o processo entre um snapshot e um backup são a mesma coisa, não significa o mesmo fim, ou seja, são demandas bem diferentes.

Veja na tabela a seguir uma melhor forma de simplificar as diferenças.

Na prática, o backup é o recurso que oferece maior segurança, integridade e eficácia ao armazenamento e requisitos de recuperação. Já o snapshot é uma estratégia de backup temporário para auxiliar em recuperações a curto prazo.
